##### 数学公式提取

发现明显局限。
部分数学符号可以保留：
```text
∑
≤
≥
```
但复杂公式结构会被破坏。如：
```text
L_t(y_t|C^S)
```
同时出现控制字符等异常 Unicode 字符：
```text
\u0013
```

结论：
> 公式内容部分保留，但公式排版和数学结构无法可靠恢复。
> 对于本项目而言影响有限，因为系统综述主要关注：理论/数据/方法名称/研究发现，而非公式本身。

## 2026-06-05

完成 section_parse.py 第一版开发，采用“正则匹配 + 规则打分 + section切分”方案；设计并实现标题模式库（英文大写标题、Title Case、阿拉伯数字标题、罗马数字标题、中文标题等）、标题候选识别函数 detect_heading_candidates()、标题打分函数 score_heading()、章节切分函数 split_sections()、批量处理与 JSON 回写流程；新增 heading_candidates、sections 等中间结果用于调试。

测试对象包括 Abbott（1995）、Heinz & Laumann（1978）、Liu & Emirbayer（2016）、King 等（2015）、O'Leary 等（2013）以及多篇中文论文。测试结果表明：
```text
中文论文标题识别效果较好，能够识别多数规范化章节标题；英文论文识别效果较差，存在大量误识别现象，包括页眉、页码、表格标题、参考文献等内容被识别为章节标题；Reference 部分未被有效截断；仅依赖文本规则和正则表达式难以稳定识别英文社会科学论文的章节结构。
```

尝试增加候选标题过滤、重复页眉删除、主模式识别（main_pattern）、Title Case 扩展等优化措施，效果有限。
> 测试结果显示问题核心不在于规则数量不足，而在于当前流程基于 cleaned_text，已经丢失 PDF 版面信息（字体大小、粗体、位置、间距等），导致章节识别缺乏关键特征。

> section_parse 流程已经跑通，JSON 读写与批量处理正常；当前版本保留作为基线版本，不再继续堆叠规则；
> 下一阶段转向研究 PyMuPDF 的版面结构提取能力，尝试保留字体、字号、block、span 等信息，并探索基于版面特征而非纯文本规则的章节识别方案。
